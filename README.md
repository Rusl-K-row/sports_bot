# Sports Bot

Telegram-бот для сбора, анализа и прогнозирования спортивных данных с целью выявления value bet (только для образовательных целей). Бот собирает статистику матчей, данные о травмах, коэффициенты букмекеров и использует машинное обучение для оценки вероятностей и поиска выгодных ставок.

## Особенности

- Сбор данных из множества источников (API‑Football, Football‑Data.org, TheSportsDB и др.)
- Парсинг информации о травмах из различных новостных источников
- Машинная обучение (логистическая регрессия) для прогнозирования исхода матча
- Обнаружение value bet на основе сравнения вероятностей модели и коэффициентов букмекеров
- Интерфейс в Telegram с интерактивными меню и обработчиками callback‑запросов
- Периодическое обновление данных и переобучение модели через планировщик
- Логирование, кэширование и расширяемая модульная архитектура

## Установка

### 1. Клонировать репозиторий

```bash
git clone git@github.com:Rusl-K-row/sports_bot.git
cd sports_bot
```

### 2. Создать виртуальное окружение (рекомендуется)

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить окружение

Скопировать примерный файл и заполнить свои значения:

```bash
cp .env.example .env
# Отредактировать .env, вставив свои токены и ключи:
#   BOT_TOKEN=ваш_токен_telegram_бота
#   API_FOOTBALL_KEY=ваш_ключ_api_football
#   FOOTBALL_DATA_KEY=ваш_ключ_football_data
#   THE_SPORTS_DB_KEY=ваш_ключ_thesportsdb   (опционально)
#   GITHUB_TOKEN=ваш_personal_access_token   (опционально, для авто‑бекапа)
#   MODEL_PATH=./data/model.joblib
#   CACHE_TTL=3600
#   ... (см. полный список в config.py)
```

### 5. Инициализировать базу данных и папки

```bash
python init_project.py
```

### 6. Запустить бота

```bash
python main.py
```

Бот запустится, инициализирует БД, запустит планировщик для периодических обновлений и начнёт слушать команды в Telegram.

В Telegram начните чат со своим ботом и отправьте `/start` – появится главное меню.

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Показать главное меню с интерактивными кнопками |
| `/help` | Показать справку и список доступных команд |
| `/status` | Показать текущее состояние бота (время работы, последнее обновление и т.д.) |
| `/matches` | Показать предстоящие матчи с коэффициентами и прогнозами |
| `/value` | Показать обнаруженные value bet (если они есть) |
| `/stats` | Показать статистику работы модели и набора данных |

Также можно пользоваться inline‑кнопками в меню для перехода между разделами (например, «📊 Посмотреть матчи», «💰 Value bet», «🩺 Отчёты о травмах», «⚙️ Настройки»).

## Планировщик

Фоновый планировщик (см. `utils/scheduler.py`) выполняет:

- Обновление данных о матчах и коэффициентов каждые `CACHE_TTL` секунд (по умолчанию 1 час)
- Обновление информации о травмах из настроенных источников
- Периодическое переобучение ML‑модели на накопленном наборе данных
- Отправку уведомлений о высокодоходных value bet (если включено)

Интервалы можно настроить в `config.py`.

## Модель

Модель в `ml/model.py` – это `LogisticRegression` (мультиклассовая), предсказывающая:

- `0` – победа хозяев (Home win)
- `1` – ничья (Draw)
- `2` – победа гостей (Away win)

Признаки включают:

- Форма команд (последние N матчей)
- История личных встреч (H2H)
- Влияние травм (кол-во и серьёзность отсутствующих игроков)
- Преимущество игры дома/в гостях
- Погодные условия (если доступны)
- Liga‑специфические константы

Модель сохраняется и загружается через `joblib` в файл, путь к которому задаётся в `MODEL_PATH` (по умолчанию `./data/model.joblib`).

## Парсер травм

Данные о травмах извлекаются из разных источников с помощью модулей в `injury_parser/`. Каждый источник реализует общий интерфейс (`BaseInjuryParser`). Сейчас включён:

- Универсальный парсер на основе регулярных выражений (`general_injuries.py`) – работает с простыми текстовыми новостями
- Можно расширять под Transfermarkt, ESPN, BBC Sport и др.

Извлечённые травмы сохраняются в таблице `matches` (поле JSON) и используются в качестве признаков в ML‑модели.

## Логирование

Вся информация о времени выполнения записывается в `logs/bot.log` (с ротацией). Логгер настраивается в `utils/logger.py` с уровнями, задаваемыми через `config.LOG_LEVEL`.

## Демонстрация и тест
